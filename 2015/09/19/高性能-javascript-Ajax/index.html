高性能 javascript系列(第二天) 理论篇之 Ajax

2015-09-19


      我总是忍不住在写点笔记之前瞎逼逼几句，不知道为什么，总感觉这是一种习惯，我觉得人总得有习惯，不管是好的还是坏的，尽管我就没得好习惯。生活要是没有了习惯，就好像你不知道该什么时候起床，起床之后是不是要洗漱。好了，下面进入正题。今天的专题之一是Ajax。Ajax是高性能javascript的基础，Ross Harmes如是说。我也不知道Ross Harmes是何许人也，是男是女？如果是女的，长得漂不漂亮？三围如何？如果是男的，我管个GG。我只关注Ajax，因为我是技术宅。

目的

      我觉得学知识和做笔记总得有个目的，总得知道自己学的玩意到底有啥用，所以我总是在瞎逼逼几句后，把目的放在笔记的开端。大家可以理解为带着目标做事效果会事半功倍，本专题的目的如下：

  • Ajax的使用。
  • 掌握服务器收发数据最快的技术。
  • 了解最为有效的数据编码格式。
  • 了解各种数据传输方法以及其对性能的影响和注意事项。
  • 牢记一些加速你的Ajax的准则。

      专家Ross harmes叫我们先牢记几个加速你的Ajax的准则：

  • 减少请求数，可通过合并javascript和CSS文件，或使用MXHR。

  • 缩短页面的加载时间，页面加载完成后，用Ajax加载次要文件。

  • 确保你的错误代码不会输出给用户，并在服务端处理结果。

  • 知道何时使用成熟的Ajax类库，何时编写自己的Ajax代码。

      上面的准则都是专家说的，专家说的要加红或加亮，我感觉专家就是喜欢装逼，说的云里雾里的，就是喜欢坑我们这些技术宅。这些准则你只管记住就好了，因为我知道你迟早得装逼，我们还是带着其他的目的进入专题。

开篇

      专家Ross harmes有这样一段描述：Ajax是高性能javascript的基础。它可以通过延迟下载体积较大的资源文件来使得页面加载速度更快。它通过一步的方式在客服端和服务端之间传输数据，从而避免页面资源一蜂窝的下载。它甚至可以只通过一次HTTP请求就获得整个页面的资源。选择合适的数据传输方式和最有效的数据格式，可以显著的改善用户和网站的交互体验。
      专家都说重要了，我们还在犹豫什么。的确，Ajax为提升你的网站的性能提供了广阔的空间，毕竟许多网站都需要大量使用异步请求。

Ajax的使用

      对了，还没说Ajax是什么呢，那Ajax是什么？专家说Ajax，全名Asynchronous Javascript And XML（异步javascript和XML），从最基本的层面来说，是一种与服务器通信而无需重新加载页面的方法。
专家说的要加红或加亮。
这意味着，通过Ajax我们可以与服务器进行通信，和服务器交换数据后，只需刷新局部页面。那么，我们如何使用Ajax？
使用Ajax需要以下几个步骤：

2：事件处理函数，处理服务器的响应结果。

onreadystatechange事件:该事件处理函数由服务器触发，而不是用户，相当于监听，监听服务器每个动作

readyState 属性表示Ajax请求的当前状态。它的值用数字代表。分别是：

0 代表未初始化。 还没有调用 open 方法
1 代表正在加载。 open 方法已被调用，但 send 方法还没有被调用
2 代表已加载完毕。send 已被调用。请求已经开始
3 代表交互中。服务器正在发送响应
4 代表完成。响应发送完毕

status 属性表示状态码，也是用数字表示，分别是：

404 没找到页面(not found)
403 禁止访问(forbidden)
500 内部服务器出错(internal service error)
200 一切正常(ok)
304 没有被修改(not modified)(服务器返回304状态，表示源文件没有被修改 )

responseText属性包含了从服务器发送的数据

每次 readyState 值的改变，都会触发 readystatechange 事件

3：打开一个连接open(method, url, asynch)

允许客户端用一个Ajax调用向服务器发送请求。

三个参数的含义如下：

method：请求类型，类似 “GET”或”POST”的字符串
url：请求路径字符串，指向所请求的服务器上的那个文件（servlet，jsp，action）
asynch：表示请求是否要异步传输，默认值为true(异步)

4：发送数据send(data)

data:向服务器发的数据，如果是get方式data为null就行，即使传了参数，服务器也收不到。如果为post方式在send(data)之前还要设置requestHeader(“Content-Type”,”application/x-www-form-urlencoded”)。

根据以上步骤，你能不能封装一个处理Ajax的函数。

数据传输

      数据传输分为请求数据和发送数据，Ajax有五种常用的技术用于向服务器请求数据。

  • XMLHttpRequest(XHR)
  • Dynamic script tag insertion 动态脚本注入
  • iframes
  • Comet
  • Multipart XHR

请求数据

      下面我们主要关注XMLHttpRequest(XHR)、动态脚本注入、Multipart XHR(MXHR)。
上面Ajax的使用例子就是用的XHR

XHR(XMLHttpRequest)

1：XHR的优点和缺点

  • XHR是目前最常用的技术，它允许异步发送和接受数据。所有的主流浏览器对它提供了完善的支持。
  • 它能精确的控制发送请求和数据接收。
  • 遗憾的是由于XHR提供了高级控制，所以浏览器对其增加了一些限制。XHR不能从外域请求数据（这句话我觉得是有条件的）。

2：对于XHR的使用，专家ROSS Harmes有以下建议：

  • 当使用XHR请求数据时，你需要在POST和GET之间做出选择。对于那些不会改变服务器状态，只会获取数据的请求，应该使用GET。经GET请求的数据会被缓存起来。
  • 只有当请求的URL加上参数的长度大于2048个字符时，才应该使用POST。过长的URL长度会导致请求的URL被截断。
    专家的建议，我总是忍不住的加亮，至于我们会不会接受这个建议，还得我们自己说了算，专家就是来制定标准的，然后默默地坑着一群人，不要被专家骗了。
    动态脚本注入
    这种请求数据的方法我也不知道是谁发明的，我至今没搞明白咋用。我在想我是不是被专家骗了。

1：动态脚本注入的优点和缺点

  • 这种方式克服了XHR的最大限制：它能跨域请求。（至于怎么跨域的，我都说了我至今没搞明白咋用）
  • 遗憾的是，动态脚本注入提供的控制是很有限的。

2：对于动态脚本注入的使用，专家Ross Harmes也给出了下面的建议：

  • 使用此项技术从你无法的服务器上请求数据时需要小心，javascript没有任何权限和访问控制的概念，因此你注入的动态脚本中的代码可能修改页面的内容甚至发送数据到第三方服务器。引入外部来源代码时，要务必小心。
    这项建议我信了，虽然不知道咋用，也不知道怎么向别人提供自己的动态脚本。从这里我觉得，专家ROSS Harmes是很谨慎的。
    Multipart XHR
          Multipart XHR是一项最新的技术。它最大的特色是允许客服端使用一个HTTP请求就可以从服务端向客服端传送多个资源。它的原理是通过在服务端将资源打包成一个由双方约定的字符串分割的长字符串并发送给客服端。客服端再javascript代码处理这个长字符串。
          我们如何使用Multipart XHR?我突然发现一个问题，专家总是喜欢取名词、下定义、秀优越，我觉得我们还是太年轻了，猜不透专家在想什么。不过菜鸟总得要有自己的活法，总得要有自己的见解。我简单的把Multipart XHR理解为：事实上Multipart XHR仍然是通过XMLHttpRequest即XHR实现的，我们知道XHR对请求数据的过程提供了高级的控制，开发人员完全可以跟踪整个过程。比如，你可以在请求中添加任何头信息和参数，也能读取任何服务器放回的头信息和数据。这就意味着，开发者可以扩展HTTP的协议，实现自己的请求和处理。这也是为什么Multipart XHR需要客服端和服务端双方做好约定的原因。
          当然这只是我的见解，我觉得如果对XHR的过程进行控制，客服端和服务端双方做好约定，就能实现Multipart XHR。至于我可不可以实现Multipar XHR，我想先参考http://techfoolery.com/Mxhr/

1：Multipart XHR的优点和缺点

  • Multipart XHR最大的特色就是允许客服端通过一次HTTP请求从服务端获取多个资源。
  • 使用Multipart XHR时的最大的缺点就是以这种方式获得的资源无法被浏览器缓存。

2：对于Multipart XHR的使用，专家ROSS Harmes有以下建议：

  • 当页面包含了大量其他地方用不到的资源，尤其是图片时，可以采用Multipart XHR。
  • 当网站已经在每个页面中使用一个独立打包的javascript或css文件已减少HTTP请求时，可以采用Multipart XHR。（因为这个时候对每个页面来说文件都是唯一的，不需要浏览器缓存）
    其实有时候专家说的还是很有道理，虽然大多时候人家饶了很多弯路讲一堆大道理，但作为菜鸟的我们还是宁可信其有，不可信其无的好。就比如说，ROSS Harmes讲了一大堆Multipart XHR后，总结了一句话：由于HTTP请求是Ajax中最大瓶颈之一，因此减少其需要的数量会对整个页面的性能有很大的影响。
    我信了，所以我加亮了。
